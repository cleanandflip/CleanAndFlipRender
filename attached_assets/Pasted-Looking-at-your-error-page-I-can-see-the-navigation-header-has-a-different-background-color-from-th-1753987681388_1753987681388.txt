Looking at your error page, I can see the navigation header has a different background color from the main content. Here's how to make it seamless:
Make Error Page Background Seamless with Navigation
Update your error page to remove the navigation bar entirely OR blend it seamlessly:
Option 1: Remove Navigation from Error Pages (Recommended)
Since error pages should be standalone, remove the navigation:
tsx// In your error-boundary.tsx, make it a full standalone page:
export function ErrorBoundary() {
  return (
    <div className="min-h-screen bg-gradient-to-b from-gray-900 via-gray-800 to-gray-900">
      {/* Remove any Layout wrapper or Navigation component */}
      {/* Just the error content without nav */}
      <div className="flex items-center justify-center px-4 min-h-screen">
        {/* Your error content */}
      </div>
    </div>
  );
}
Option 2: Make Navigation Background Transparent on Error Pages
If you must keep the navigation, update your Layout component:
tsx// In your Layout component:
import { useLocation } from 'react-router-dom';

export function Layout() {
  const location = useLocation();
  const isErrorPage = location.pathname.includes('404') || location.pathname.includes('error');
  
  return (
    <div className="min-h-screen bg-gradient-to-b from-gray-900 via-gray-800 to-gray-900">
      <nav className={isErrorPage ? "bg-transparent" : "bg-gray-900/95 backdrop-blur-sm"}>
        {/* Your navigation content */}
      </nav>
      {/* Rest of layout */}
    </div>
  );
}
Option 3: Update Navigation Component Directly
In your navigation.tsx file, make the background match:
tsx// Change from:
<nav className="bg-gray-800 border-b border-gray-700">

// To:
<nav className="bg-transparent">
  {/* Or if you need some definition: */}
  <nav className="bg-gray-900/50 backdrop-blur-sm">
Option 4: Create a Separate Error Layout
Create a minimal layout just for errors:
tsx// error-layout.tsx
export function ErrorLayout({ children }) {
  return (
    <div className="min-h-screen bg-gradient-to-b from-gray-900 via-gray-800 to-gray-900">
      {/* Minimal header with just logo */}
      <div className="absolute top-0 left-0 p-6">
        <a href="/" className="flex items-center">
          <img src="/logo.png" alt="Clean & Flip" className="h-10" />
        </a>
      </div>
      {children}
    </div>
  );
}
The cleanest approach is Option 1 - error pages typically don't need full navigation since they're dead ends. Just include the logo as a home link and the essential error information.
