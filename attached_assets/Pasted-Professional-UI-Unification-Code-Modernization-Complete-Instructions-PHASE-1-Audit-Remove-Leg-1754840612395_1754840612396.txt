Professional UI Unification & Code Modernization - Complete Instructions
PHASE 1: Audit & Remove Legacy Code
Step 1: Find and Remove Deprecated Code
Instructions for Replit:

Search entire codebase for:

Inline styles (style=) - move to Tailwind classes
Hard-coded pixel values - replace with rem/Tailwind
!important CSS declarations - refactor specificity
Commented-out code blocks - delete if older than 1 month
Console.log statements - remove from production
Unused imports - remove all
Dead components (not imported anywhere) - delete files


Remove these deprecated patterns:

className="w-[423px]" → Use standard Tailwind widths
style={{ marginTop: '20px' }} → Use Tailwind spacing
Mixed units (px, rem, em) → Standardize to rem
Custom CSS files except globals → Migrate to Tailwind




PHASE 2: Establish Measurement System
Step 2: Create Core Spacing Scale
File: client/src/config/dimensions.ts
Create a single source of truth for ALL measurements:
Base unit: 4px (0.25rem)
Scale: 4, 8, 12, 16, 20, 24, 32, 40, 48, 56, 64, 80, 96

Spacing classes to use everywhere:
- space-1 (4px) - Tiny gaps
- space-2 (8px) - Icon gaps, small margins  
- space-3 (12px) - Form field gaps
- space-4 (16px) - Standard gap, card padding mobile
- space-5 (20px) - Section gaps mobile
- space-6 (24px) - Card padding desktop
- space-8 (32px) - Large gaps
- space-10 (40px) - Section spacing mobile
- space-12 (48px) - Major section gaps
- space-16 (64px) - Section spacing desktop
- space-20 (80px) - Page sections
Step 3: Component Height Standardization
Every interactive element must use these exact heights:
Interactive Element Heights:
- xs: h-7 (28px) - Tags, chips only
- sm: h-8 (32px) - Secondary actions
- md: h-10 (40px) - DEFAULT for all inputs/buttons
- lg: h-12 (48px) - Primary CTAs
- xl: h-14 (56px) - Hero buttons only

Text Input Specific:
- ALL text inputs: h-10 (40px)
- ALL textareas: min-h-[80px]
- ALL selects: h-10 (40px)

Fixed Heights:
- Navigation bar: h-16 (64px)
- Mobile nav: h-14 (56px)  
- Tab bars: h-12 (48px)
- Table rows: h-13 (52px)
- Sidebar items: h-11 (44px)

PHASE 3: Component-by-Component Unification
Step 4: Button Standardization
Update EVERY button in the application:

Find all button instances:

Search: <button, <Button, onClick=
Files to check: Every .tsx file


Apply these rules to EACH button:
Primary buttons (main actions):
- Height: h-10 (40px) desktop, h-12 (48px) mobile
- Padding: px-4 desktop, px-6 mobile
- Font: text-base (16px)
- Border radius: rounded-lg (8px)
- Min width: min-w-[120px]

Secondary buttons:
- Height: h-10 (40px) always
- Padding: px-3
- Font: text-sm (14px)
- Same border radius

Icon-only buttons:
- Size: w-10 h-10 (40px square)
- Icon size: w-5 h-5 (20px)
- Border radius: rounded-lg

Text buttons:
- Height: h-10
- No background
- Padding: px-2

Remove these patterns:

Variable heights across pages
Inconsistent padding
Mixed border radius
Buttons smaller than 32px height
Buttons without hover states



Step 5: Input Field Unification
Standardize EVERY input, select, and textarea:

Find all inputs:

Search: <input, <Input, <select, <textarea
Check all form components


Apply uniform styling:
All text inputs:
- Height: h-10 (40px) EXACTLY
- Padding: px-3 (12px horizontal)

- Border: border (1px)
- Border radius: rounded-lg (8px)

All selects:

All textareas:
- Min height: min-h-[80px]

Labels:
- Font: text-sm (14px)
- Margin bottom: mb-2 (8px)
- Font weight: font-medium

Error messages:
- Font: text-sm (14px)
- Margin top: mt-1 (4px)
- Consistent error color

PHASE 5: Interaction States
Step 12: Standardize Interactive Feedback
Apply to all interactive elements:
Hover states:
- Buttons: brightness-110 or darker shade
- Cards: shadow-md → shadow-lg, translateY(-2px)
- Links: underline or opacity change
- Duration: transition-all duration-200

Focus states:
- Outline: ring-2 ring-offset-2
- No outline-none without alternative
- Tab navigation must be visible

Active/pressed states:
- Buttons: scale-95
- Cards: shadow-sm
- Immediate feedback

Disabled states:
- Opacity: opacity-50
- Cursor: cursor-not-allowed
- No hover effects

Loading states:
- Skeleton height matches final content
- Shimmer animation consistent
- Minimum 200ms display

Step 13: Icon Size Standardization

Icon size rules:

PHASE 8: Component Cleanup
Step 15: Remove/Consolidate Duplicate Components
Instructions for finding and merging:

Find duplicates:

Multiple button components → Use one Button.tsx
Multiple card components → Use one Card.tsx
Multiple modal components → Use one Modal.tsx
Custom inputs → Use one Input.tsx


Consolidation rules:

Keep the most feature-complete version
Add variants for different styles
Remove component-specific styling
Use composition over duplication


Files to delete:

Any component with "old", "backup", "v2" in name
Unused components (0 imports)
Test components in production




PHASE 9: Mobile-First Refactor
Step 16: Update All Components for Mobile-First
Refactor order for every component:
1. Write mobile styles first (no prefix)
2. Add tablet styles (sm: prefix)  
3. Add desktop styles (lg: prefix)

Example pattern:
- padding: p-4 sm:p-5 lg:p-6
- text: text-base lg:text-lg
- grid: grid-cols-1 sm:grid-cols-2 lg:grid-cols-4

Remove:
- Desktop-first patterns (unprefixed large values)
- Hidden on mobile critical features
- Fixed widths without responsive alternatives

PHASE 10: Performance & Code Quality
Step 17: Optimization Instructions

Import optimization:

Group imports: React, libraries, components, utils, types
Remove unused imports
Use index exports for components


Component optimization:

Add React.memo to list items
Extract static arrays/objects outside components
Use proper keys in lists


CSS optimization:

Remove unused Tailwind classes
No arbitrary values [123px]
Use Tailwind's built-in classes only


Image optimization:

Add loading="lazy" to all images
Add aspect-ratio containers
Use proper alt text
